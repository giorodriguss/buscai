02
Tecnologias
Frontend
Flutter
Interface do usuário, feed de postagens, perfis e cache de imagens. Um único código para Android e iOS.
Backend
NestJS
Controllers, services, regras de negócio e filtros. Hospedado no Railway com servidor persistente.
Banco de dados
PostgreSQL
Banco relacional robusto via Supabase para usuários, postagens e avaliações.
Storage
Supabase
Fotos de perfil e portfólio. Autenticação com Row Level Security na mesma plataforma.
Comunicação
WhatsApp
Deep Link via wa.me. Zero custo operacional, zero intermediário entre morador e prestador.
Implantação
Railway
Infraestrutura cloud com custo progressivo. Escala conforme o crescimento do sistema.
03
Arquitetura do Sistema
Frontend
Flutter — Web + App
Interface do usuário, feed de postagens, perfis de prestadores e cache de imagens. Distribuição simultânea para Android e iOS.
Requisições JSON / HTTP
Backend · Railway
API NestJS
Controllers, services, regras de negócio e filtros. O frontend nunca acessa o banco diretamente — toda comunicação passa pela API.
Consulta SQL · URLs de imagens
Banco de dados
PostgreSQL · Supabase
Tabelas: usuários, postagens, avaliações.
Storage
Supabase Storage
Fotos de perfil e imagens de portfólio.
